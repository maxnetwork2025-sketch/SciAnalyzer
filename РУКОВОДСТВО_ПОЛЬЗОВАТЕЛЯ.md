## 1. Что такое SciAnalyzer

**SciAnalyzer** — настольное приложение для автоматизированного поиска, анализа и систематизации научных публикаций. Оно объединяет в одном окне семь научных баз данных, встроенный ИИ-помощник на базе Ollama и инструменты для работы с документами.

**Ключевые возможности:**

| Возможность | Описание |
|---|---|
| Поиск | 7 источников одновременно — российские и международные |
| Скачивание | PDF и TXT прямо в библиотеку |
| Реферирование | ИИ создаёт краткое резюме любой статьи |
| Перевод | Автоматический перевод статьи на русский язык |
| Сравнение | Анализ сходства двух документов |
| Семантический поиск | Поиск по смыслу, а не по ключевым словам |
| Документы | Генерация отчётов и справок по шаблонам |

## 2. Установка

### Системные требования

| Параметр | Минимум | Рекомендуется |
|---|---|---|
| ОС | Windows 10 (64-bit) | Windows 11 |
| RAM | 8 ГБ | 16 ГБ |
| Место на диске | 5 ГБ | 10 ГБ |
| CPU | 4 ядра | 8 ядер |
| Интернет | Нужен для поиска | — |

> Отдельная видеокарта **не требуется** — ИИ-модель работает на процессоре.

### Шаги установки

1. Запустите файл `SciAnalyzer_Setup.exe`.
2. При появлении запроса Защитника Windows нажмите **«Подробнее» → «Всё равно выполнить»**.
3. Следуйте мастеру установки (нажимайте **Далее**).
4. Опционально создайте ярлык на рабочем столе.
5. По окончании нажмите **Готово** — приложение запустится автоматически.

> Все компоненты (включая ИИ-модель) установлены внутри пакета. Дополнительных загрузок не требуется.

## 3. Первый запуск и вход

### Вход по умолчанию

При первом запуске используйте учётную запись администратора:

```
Логин:  admin
Пароль: admin
```

> **Важно:** сразу смените пароль администратора в разделе **Настройки → Аккаунт**.

### Установка пароля для нового пользователя

Если администратор создал вам учётную запись, при первом входе система попросит задать собственный пароль. Требования к паролю:

- Минимум 4 символа
- Допускается любая раскладка

## 5. Поиск статей

### Источники данных

SciAnalyzer ищет одновременно в семи базах:

| База | Язык | Область |
|---|---|---|
| **arXiv** | EN | Физика, математика, CS, биология |
| **PubMed** | EN | Медицина и биомедицина |
| **Semantic Scholar** | EN | Все области науки |
| **КиберЛенинка** | RU | Все области науки |
| **СибАК** | RU | Все области науки |
| **Молодой учёный** | RU | Все области науки |
| **eLibrary** | RU | Все области науки |

### Как выполнить поиск

1. Откройте вкладку **Поиск**.
2. Введите запрос в поле (тема, ключевые слова или имя автора).
3. Выберите режим: **По теме** или **По автору**.
4. Укажите максимальное количество результатов (5 / 10 / 20 / 50).
5. Отметьте галочками нужные источники.
6. Нажмите **Найти**.

### Работа с результатами

Каждая найденная статья отображается карточкой с цветовой меткой источника. Для каждой статьи доступны действия:

| Кнопка | Действие |
|---|---|
| **Открыть** | Открыть страницу статьи в браузере |
| **Скачать** | Скачать PDF или TXT в библиотеку |
| **Резюме** | ИИ создаёт краткое изложение (3–7 предложений) |
| **Перевод** | Перевести на русский язык |

> **Подсказка:** чтобы сохранить статью для дальнейшей работы, обязательно нажмите **Скачать** — иначе она не попадёт в библиотеку.

### История поиска

Приложение автоматически сохраняет историю запросов. Нажмите на поле поиска — появится выпадающий список последних запросов для быстрого повтора.

## 6. Библиотека

Библиотека содержит все скачанные вами статьи, хранящиеся локально на компьютере.

### Панель фильтров (левая колонка)

- **Источник** — фильтр по базе данных (arXiv, PubMed и др.)
- **Поиск по названию** — фильтр по тексту в заголовке
- **Семантический поиск** — поиск по смыслу запроса (требует предварительного вычисления эмбеддингов)

### Действия со статьёй

Выберите статью в списке и используйте кнопки панели:

| Кнопка | Действие |
|---|---|
| **Открыть** | Открыть документ во встроенном просмотрщике |
| **Резюме** | ИИ-реферат статьи |
| **Перевод** | Перевод на русский язык |
| **Эмбеддинг** | Вычислить вектор для семантического поиска |
| **Удалить** | Убрать статью из библиотеки |

### Встроенный просмотрщик

Позволяет читать PDF и DOCX прямо в приложении без запуска сторонних программ. Поддерживает прокрутку и навигацию по страницам.

### Семантический поиск

Семантический поиск находит статьи, близкие **по смыслу**, а не только по совпадению слов.

**Для активации:**
1. Выделите нужные статьи в библиотеке.
2. Нажмите **Эмбеддинг** для каждой статьи (или для всех сразу).
3. Введите запрос в поле семантического поиска и нажмите кнопку поиска.

> Вычисление эмбеддингов занимает несколько секунд на статью и выполняется один раз.

## 7. Сравнение документов

Вкладка **Сравнение** позволяет проверить два документа на смысловое сходство — полезно при проверке на плагиат или при анализе пересечений источников.

### Форматы файлов

Поддерживаются: `.pdf`, `.docx`, `.txt`

### Как сравнить документы

1. Откройте вкладку **Сравнение**.
2. Добавьте **Документ A**: нажмите **Выбрать файл** или **Из библиотеки**.
3. Аналогично добавьте **Документ B**.
4. Нажмите **Сравнить**.

### Результат сравнения

Приложение показывает:
- **Процент сходства** (0–100 %)
- **Общие фрагменты** с подсветкой совпадающих частей
- **Вывод**: уникальные и совпадающие блоки текста

| Сходство | Интерпретация |
|---|---|
| 0–20 % | Документы существенно различаются |
| 20–50 % | Частичное пересечение тематики |
| 50–80 % | Значительное сходство, требует внимания |
| 80–100 % | Документы практически идентичны |

## 8. Генерация документов

Вкладка **Документы** позволяет создавать оформленные Word-файлы на основе готовых шаблонов.

### Как создать документ

1. Откройте вкладку **Документы**.
2. В левой панели выберите шаблон из списка.
3. Заполните поля в правой части (название, автор, дата и т.д.).
4. Нажмите **Создать документ**.
5. Готовый файл `.docx` сохранится в папку `СозданныеДокументы` и откроется автоматически.

### Добавление собственных шаблонов

1. Откройте проводник и перейдите в:
   ```
   %APPDATA%\SciAnalyzer\ШаблоныДокументов\
   ```
2. Скопируйте ваш `.docx`-файл в эту папку.
3. В приложении нажмите кнопку обновления (⟳) над списком шаблонов.

> Шаблоны используют синтаксис **Jinja2**. Переменные записываются в двойных фигурных скобках: `{{ название }}`, `{{ автор }}`, `{{ дата }}`.

## 10. Настройки и управление пользователями

### Смена пароля

1. Откройте **Настройки → Аккаунт**.
2. Нажмите **Сменить пароль**.
3. Введите новый пароль дважды и подтвердите.

### Управление пользователями (только для администратора)

Администратор видит дополнительный раздел **Пользователи**, где можно:

| Действие | Описание |
|---|---|
| **Добавить пользователя** | Создать новый аккаунт (пароль задаётся при первом входе) |
| **Удалить пользователя** | Безвозвратно удалить аккаунт вместе со всеми его статьями |
| **Сбросить пароль** | Сбросить пароль выбранного пользователя |

> Учётную запись `admin` удалить невозможно.

### Данные приложения

Все пользовательские данные хранятся в:
```
%APPDATA%\SciAnalyzer\
├── scianalyzer.db          — база данных
├── НайденныеСтатьи\        — скачанные PDF и TXT
├── СозданныеДокументы\     — готовые документы
└── ШаблоныДокументов\      — шаблоны Word
```

## 11. Искусственный интеллект в SciAnalyzer

SciAnalyzer использует локальную ИИ-модель **Qwen 2.5 (3B)**, работающую через сервис **Ollama**. Вся обработка происходит **на вашем компьютере** — данные никуда не отправляются.

### Что умеет ИИ

| Функция | Описание |
|---|---|
| **Реферирование** | Выделяет 3–4 ключевых утверждения из каждого раздела, затем объединяет в связное резюме на 5–7 предложений |
| **Перевод** | Переводит текст на русский язык |
| **Семантический поиск** | Создаёт векторное представление текста для поиска по смыслу |
| **Извлечение метаданных** | Автоматически определяет название, авторов и год публикации |

### Время обработки

| Операция | Примерное время |
|---|---|
| Реферирование короткой статьи (до 5000 символов) | 15–30 сек |
| Реферирование длинной статьи (более 20 000 символов) | 1–3 мин |
| Перевод страницы | 20–60 сек |
| Вычисление эмбеддинга | 5–15 сек |

> Время зависит от мощности процессора. На первый запрос после старта приложения уходит немного больше времени — модель разогревается.

### Если ИИ не отвечает

1. Подождите 30 секунд после запуска приложения — Ollama стартует в фоне.
2. Повторите операцию — первый запрос всегда чуть медленнее.
3. Если ошибка повторяется — перезапустите приложение.

## 12. Часто задаваемые вопросы

**Q: Приложение долго запускается в первый раз — это нормально?**
Да. При первом запуске Ollama инициализирует ИИ-модель (~1.8 ГБ). Последующие запуски занимают 5–10 секунд.

---

**Q: Поиск по eLibrary не работает или выдаёт мало результатов.**
eLibrary имеет ограниченный открытый доступ. Часть статей доступна только авторизованным пользователям на сайте. SciAnalyzer получает только открытые результаты.

---

**Q: Можно ли работать без интернета?**
ИИ-функции (реферирование, перевод, семантический поиск) работают полностью офлайн. Для поиска новых статей и скачивания PDF требуется интернет-соединение. Уже скачанные статьи доступны всегда.

---

**Q: Где хранятся скачанные статьи?**
В папке `%APPDATA%\SciAnalyzer\НайденныеСтатьи\`. Вы можете открыть её через Проводник Windows.

---

**Q: Как сделать резервную копию библиотеки?**
Скопируйте папку `%APPDATA%\SciAnalyzer\` целиком. Для восстановления — вставьте её на место.

---

**Q: Можно ли использовать приложение нескольким пользователям на одном компьютере?**
Да. Каждый пользователь входит под своим логином и видит только свои статьи. Администратор управляет аккаунтами через раздел **Настройки → Пользователи**.

---

**Q: Приложение показывает «Ollama не запустилась».**
Убедитесь, что файл `ollama.exe` присутствует в папке установки (`%LOCALAPPDATA%\SciAnalyzer\`). Если файл исчез — переустановите приложение.
